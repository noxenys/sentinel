# 贡献指南 / Contributing Guide

欢迎为 Sentinel 项目做出贡献！我们感谢所有形式的贡献，包括报告问题、提交代码、改进文档等。

We welcome contributions to the Sentinel project! We appreciate all forms of contributions, including reporting issues, submitting code, improving documentation, etc.

## 📋 报告问题 / Reporting Issues

### 中文
- 使用清晰、描述性的标题
- 提供详细的复现步骤
- 包含环境信息（浏览器、操作系统等）
- 附上相关日志、截图或错误信息
- 如果是功能请求，请描述使用场景和预期效果

### English
- Use clear, descriptive titles
- Provide detailed reproduction steps
- Include environment information (browser, OS, etc.)
- Attach relevant logs, screenshots, or error messages
- For feature requests, describe usage scenarios and expected outcomes

## 💻 提交代码 / Submitting Code

### 中文
1. **Fork 项目仓库**
2. **创建功能分支**：`git checkout -b feature/amazing-feature`
3. **提交更改**：使用清晰的提交信息
4. **推送到分支**：`git push origin feature/amazing-feature`
5. **创建 Pull Request**

### English
1. **Fork the project repository**
2. **Create a feature branch**: `git checkout -b feature/amazing-feature`
3. **Commit your changes**: Use clear commit messages
4. **Push to the branch**: `git push origin feature/amazing-feature`
5. **Create a Pull Request**

## 🎯 代码规范 / Code Standards

### 中文
- 遵循 JavaScript 最佳实践
- 保持代码注释清晰、有用
- 确保代码格式一致
- 添加适当的错误处理
- 测试你的更改

### English
- Follow JavaScript best practices
- Maintain clear and useful code comments
- Ensure consistent code formatting
- Add appropriate error handling
- Test your changes

## 📚 文档改进 / Documentation Improvements

### 中文
- 保持文档与代码同步
- 使用清晰、简洁的语言
- 提供示例代码和截图
- 检查拼写和语法错误

### English
- Keep documentation synchronized with code
- Use clear and concise language

### English

git clone https://github.com/your-username/sentinel.git

npm install

## 🧪 测试 / Testing

### 中文
- 在提交前测试你的更改
- 确保现有功能不受影响
- 测试不同浏览器和设备
- 验证错误处理逻辑

### English
- Test your changes before submitting
- Ensure existing functionality is not affected
- Test across different browsers and devices
- Verify error handling logic

## 📝 Pull Request 流程 / Pull Request Process

### 中文
1. **描述更改**：清晰说明 PR 的目的和内容
2. **关联问题**：如果解决特定问题，请关联 issue
3. **代码审查**：准备好根据反馈进行修改
4. **持续集成**：确保所有检查通过
5. **合并批准**：等待维护者批准合并

### English
1. **Describe changes**: Clearly explain the purpose and content of the PR
2. **Link issues**: If resolving a specific issue, link the issue
3. **Code review**: Be prepared to make changes based on feedback
4. **Continuous integration**: Ensure all checks pass
5. **Merge approval**: Wait for maintainer approval to merge

## 🏷️ 提交信息规范 / Commit Message Guidelines

### 中文
使用约定式提交格式：
```
<类型>[可选的作用域]: <描述>

[可选的正文]

[可选的脚注]
```

**类型**: feat, fix, docs, style, refactor, test, chore

### English
Use conventional commit format:
```
<type>[optional scope]: <description>

[optional body]

[optional footer]
```

**Types**: feat, fix, docs, style, refactor, test, chore
